ping google.com
Sends ICMP echo requests to the target host and waits for replies. Helps verify internet connectivity and measure latency (time delay). If replies are received, the host is reachable; otherwise, the host or network may be down.

ifconfig
Displays details of active and inactive interfaces such as eth0, wlan0, etc. Useful for identifying your systemâ€™s current IP address and verifying interface connectivity.

traceroute google.com
Helps diagnose routing problems and identify where a connection fails. Each line in the output shows a hop (router) along the path with response time.

netstat -r
Displays the routing table, showing how packets are forwarded. Columns include Destination, Gateway, Genmask, and Interface. Helps verify proper routing and gateway configuration.

nslookup google.com
Confirms that DNS (Domain Name System) is functioning correctly. Returns both the domain name and the associated IP address

dig google.com
Performs detailed DNS lookups and shows query time, server used, and record type (A, MX, NS, etc.).
Helps analyze DNS resolution issues and verify domain-to-IP mapping.

arp -a
Displays the ARP (Address Resolution Protocol) table, listing IP addresses mapped to MAC addresses on the local network.
Useful for identifying connected devices and troubleshooting LAN communication issues.

netstat -tuln
Lists all active TCP and UDP ports and their listening services.
Helps detect open ports, confirm if a server/service (like FTP or SSH) is running, and check for unwanted connections.
